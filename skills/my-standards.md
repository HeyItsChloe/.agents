# My Coding Standards

These are my personal coding preferences that apply across all my repositories.

## General Philosophy

- **Readability first** - Code is read more than written
- **Keep it simple** - Avoid over-engineering
- **Test the happy path** - Don't mock everything

## Formatting

- 2 spaces for indentation in JS/TS
- 4 spaces for Python
- Always use trailing commas
- Single quotes in JS, double quotes in Python

## Naming

- Descriptive variable names (no single letters except loop counters)
- Boolean variables start with `is`, `has`, `should`
- Functions describe actions: `getUserById`, `validateEmail`

## Git Practices

- Commit messages: imperative mood ("Add feature" not "Added feature")
- Small, focused commits
- Branch naming: `feature/description` or `fix/description`

## Comments

- Only comment "why", not "what"
- Delete commented-out code
- TODO comments must have a name and date
