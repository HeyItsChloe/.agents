# Email Style

This skill defines email composition preferences for the Email Agent.

## Signature

Always end emails with:
```
Best,
Chloe
```

## Greeting Style

Vary greetings based on context:

| Context | Greeting |
|---------|----------|
| Professional/formal | "Hello," or "Hi [Name]," |
| Casual/known contact | "Hey," or "Hey [Name]," |
| Quick follow-up | No greeting needed, just continue thread |
| First contact | "Hi [Name]," |

## Tone Guidelines

**Context-dependent tone:**

1. **Professional emails** (recruiters, business contacts, official matters):
   - Polite and professional
   - Complete sentences
   - Clear and direct

2. **Casual emails** (friends, familiar colleagues, informal):
   - Relaxed tone
   - Can use contractions
   - Brief is fine

3. **Technical emails** (code reviews, bug reports, technical discussions):
   - Direct and specific
   - Include relevant details
   - Skip pleasantries, get to the point

## Email Structure

1. **Greeting** (context-appropriate)
2. **Main content** (succinct, actionable)
3. **Signature** (Best, Chloe)

## Examples

**Professional:**
```
Hi Sarah,

Thanks for reaching out. I'm available Thursday afternoon for a call.

Best,
Chloe
```

**Casual:**
```
Hey!

That works for me. See you at 3.

Best,
Chloe
```

**Technical:**
```
The issue is in line 42 of api.ts - the null check is missing.

Here's the fix:
[code snippet]

Best,
Chloe
```

## What NOT to Do

- Don't use overly formal language like "Dear Sir/Madam"
- Don't include unnecessary pleasantries ("I hope this email finds you well...")
- Don't write lengthy emails when brief ones work
- Don't forget the signature
