# Environment Setup — {{REPO_NAME}}

Writes `{{CONFIG_FILE}}` from OpenHands secrets before run commands.

## Detected Config

- **Config file:** `{{CONFIG_FILE}}`
- **Required vars:** `{{REQUIRED_ENV_VARS}}`
- **Optional vars:** `{{OPTIONAL_ENV_VARS}}`

## Process

### Step 1: Check for Existing Config Template

```bash
# Check if .env.example or similar template exists
ls -la .env* 2>/dev/null
cat .env.example 2>/dev/null || cat .env.sample 2>/dev/null || echo "No template found"
```

### Step 2: Read OpenHands Secrets

```bash
# OpenHands injects secrets as environment variables
# Available secrets should be listed in OpenHands settings
# For this template, we use common patterns:
echo "Checking for common secrets..."
env | grep -E "API_KEY|TOKEN|SECRET|DB_|DATABASE|PASSWORD" | sed 's/=.*/=***/' || echo "No secrets found in environment"
```

### Step 3: Write Config File

```bash
# Create .env file with secrets
cat > {{CONFIG_FILE}} << 'EOF'
# Environment Configuration
# Generated by pipeline-generator automation

# Database
{{DB_CONNECTION_LINE}}

# API Keys
{{API_KEYS_LINE}}

# Application Settings
{{APP_SETTINGS_LINE}}
EOF

# Alternative: Write directly with expected secrets
cat > {{CONFIG_FILE}} << 'EOF'
# Environment Configuration
# Generated by pipeline-generator automation

# Database configuration
{{DB_CONNECTION}}

# API configuration
{{API_KEY}}

# Application settings
{{APP_SECRET}}
EOF
```

### Step 4: Verify Config File

```bash
# Ensure config file was created
ls -la {{CONFIG_FILE}}
head -20 {{CONFIG_FILE}}
```

## Common Patterns

### Node.js (package.json based)
```
# .env
DB_CONNECTION=mongodb+srv://user:pass@cluster.mongodb.net/db
API_KEY=your_api_key_here
PORT=3000
NODE_ENV=development
```

### Python (requirements.txt based)
```
# .env or settings.py
DATABASE_URL=postgresql://user:pass@localhost/db
SECRET_KEY=your_secret_key_here
DEBUG=True
```

### Ruby/Rails
```
# .env
DATABASE_URL=postgresql://user:pass@localhost/myapp_dev
REDIS_URL=redis://localhost:6379
SECRET_KEY_BASE=your_secret_key_base
```

### Go
```
# .env or config.yaml
DATABASE_URL=postgres://user:pass@localhost/db
API_KEY=your_api_key
LOG_LEVEL=debug
```

## Error Handling

If config file cannot be written:
1. Log warning to `/tmp/env-setup-warnings.log`
2. Continue with pipeline (some apps have default configs)
3. Document missing config in PR comment
