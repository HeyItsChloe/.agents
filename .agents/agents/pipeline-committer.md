# Pipeline Committer Agent

Commits all generated files to the target repository and creates a PR with the audit summary.

## Inputs

- Pipeline spec: `/tmp/pipeline-spec-{REPO_NAME}.json`
- Target repo: `/tmp/{REPO_NAME}`
- Audit JSON: `/tmp/audit-{REPO_NAME}.json`

## Process

### Step 1: Read Pipeline Spec

```bash
cat /tmp/pipeline-spec-{REPO_NAME}.json | jq .
OWNER=$(jq -r '.owner' /tmp/pipeline-spec-{REPO_NAME}.json)
REPO_NAME=$(jq -r '.repo_name' /tmp/pipeline-spec-{REPO_NAME}.json)
```

### Step 2: Configure Git

```bash
cd /tmp/{REPO_NAME}
git config user.name "openhands" 2>/dev/null || true
git config user.email "openhands@all-hands.dev" 2>/dev/null || true
git init 2>/dev/null || true
git remote add origin "https://oauth2:${GITHUB_TOKEN}@github.com/${OWNER}/${REPO_NAME}.git" 2>/dev/null || true
```

### Step 3: Create Branch

```bash
cd /tmp/{REPO_NAME}
git checkout -b "pipeline-gen/$(date +%Y%m%d-%H%M%S)"
```

### Step 4: Stage All Generated Files

```bash
cd /tmp/{REPO_NAME}
git add .agents/agents/ .agents/skills/ .agents/automations/
git status
```

### Step 5: Generate Commit Message

```bash
# Build commit message from audit summary
TECH_STACK=$(jq -r '.audit_summary.tech_stack // "Unknown"' /tmp/pipeline-spec-{REPO_NAME}.json)
AGENT_COUNT=$(jq -r '.audit_summary.total_agents // 5' /tmp/pipeline-spec-{REPO_NAME}.json)
SKILL_COUNT=$(jq -r '.audit_summary.total_skills // 3' /tmp/pipeline-spec-{REPO_NAME}.json)
CONDITIONAL=$(jq -r '.audit_summary.conditional_added // [] | join(", ")' /tmp/pipeline-spec-{REPO_NAME}.json)

COMMIT_MSG="feat: add autonomous dev pipeline for {REPO_NAME}

Generated by pipeline-generator automation.

Tech stack: {TECH_STACK}
Agents: {AGENT_COUNT}
Skills: {SKILL_COUNT}
Conditional: {CONDITIONAL:-none}

- ticket-planner: reads issues, writes implementation plans
- code-implementer: creates branches, writes code
- tester: writes missing tests
- pr-reviewer: self-reviews, posts comments
- ticket-manager: creates PRs linked to issues
- env-setup: configures environment from secrets
- ci-monitor: polls CI checks, retries on failure
- whatsapp-notifier: sends review notifications

Trigger: label an issue 'ready-to-implement' to start the pipeline."
```

### Step 6: Commit Changes

```bash
cd /tmp/{REPO_NAME}
git add -A
git commit -m "{COMMIT_MSG}"
```

### Step 7: Push Branch

```bash
cd /tmp/{REPO_NAME}
git push -u origin HEAD
```

### Step 8: Create PR

```bash
cd /tmp/{REPO_NAME}

# Get audit summary for PR body
AUDIT_SUMMARY=$(cat << 'SUMMARY'
## Pipeline Generator — Audit Summary

### Tech Stack Detected
- **Language:** {LANGUAGE}
- **Framework:** {FRAMEWORK}
- **Package Manager:** {PACKAGE_MANAGER}

### Project Structure
- **Source dirs:** {SOURCE_DIRS}
- **Test dirs:** {TEST_DIRS}
- **Config files:** {CONFIG_FILES}

### Commands
- **Install:** {INSTALL_CMD}
- **Test:** {TEST_CMD}
- **Build:** {BUILD_CMD}

### CI/CD
- **Type:** {CI_TYPE}
- **Workflows:** {WORKFLOWS}

### Generated Pipeline

| Component | Type | Count |
|-----------|------|-------|
| Agents | Universal + Conditional | {AGENT_COUNT} |
| Skills | Universal + Conditional | {SKILL_COUNT} |

#### Agents Generated
{AGENT_LIST}

#### Skills Generated
{SKILL_LIST}

### Conditional Components Added
{CONDITIONAL_LIST}

### Next Steps

1. **Review the PR** — all files are in `.agents/` directory
2. **Register secrets** in OpenHands → Settings → Secrets:
   - `GITHUB_TOKEN` (for all GitHub operations)
   - Any repo-specific secrets (API keys, DB connections, etc.)
3. **Label an issue** with `ready-to-implement` to trigger the pipeline
4. **Review PR** when automation completes and WhatsApp notification arrives

### What the Pipeline Will Never Do

- ❌ Merge to `main`
- ❌ Deploy to production
- ❌ Modify production environment variables
- ❌ Upload to app stores or package registries
SUMMARY

# Replace placeholders
sed -i.bak \
  -e 's/{LANGUAGE}/'"$(jq -r '.tech_stack.language // "unknown"' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{FRAMEWORK}/'"$(jq -r '.tech_stack.framework // "unknown"' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{PACKAGE_MANAGER}/'"$(jq -r '.tech_stack.package_manager // "npm"' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{SOURCE_DIRS}/'"$(jq -r '.project_structure.source_dirs | join(", ")' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{TEST_DIRS}/'"$(jq -r '.project_structure.test_dirs | join(", ")' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{CONFIG_FILES}/'"$(jq -r '.project_structure.config_files | join(", ")' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{INSTALL_CMD}/'"$(jq -r '.commands.install // "npm install"' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{TEST_CMD}/'"$(jq -r '.commands.test // "npm test"' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{BUILD_CMD}/'"$(jq -r '.commands.build // "npm run build"' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{CI_TYPE}/'"$(jq -r '.ci_system.type // "github-actions"' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{WORKFLOWS}/'"$(jq -r '.ci_system.workflows | join(", ")' /tmp/audit-{REPO_NAME}.json)"'/g' \
  -e 's/{AGENT_COUNT}/'"$(jq -r '.audit_summary.total_agents // 5' /tmp/pipeline-spec-{REPO_NAME}.json)"'/g' \
  -e 's/{SKILL_COUNT}/'"$(jq -r '.audit_summary.total_skills // 3' /tmp/pipeline-spec-{REPO_NAME}.json)"'/g' \
  -e 's/{AGENT_LIST}/'"$(jq -r '.agents.universal[].name, .agents.conditional[].name' /tmp/pipeline-spec-{REPO_NAME}.json | sed 's/^/- /' | tr '\n' '\n')"'/g' \
  -e 's/{SKILL_LIST}/'"$(jq -r '.skills.universal[].name, .skills.conditional[].name' /tmp/pipeline-spec-{REPO_NAME}.json | sed 's/^/- /' | tr '\n' '\n')"'/g' \
  -e 's/{CONDITIONAL_LIST}/'"$(jq -r '.audit_summary.conditional_added // [] | map("+ \(.)") | join("\n")' /tmp/pipeline-spec-{REPO_NAME}.json)"'/g' \
  /tmp/pr-body.md

# Create the PR
gh pr create \
  --repo "$OWNER/$REPO_NAME" \
  --title "feat: add autonomous dev pipeline for $REPO_NAME" \
  --body "$(cat /tmp/pr-body.md)" \
  --base main \
  --draft
```

### Step 9: Verify

```bash
echo "PR created:"
gh pr view --repo "$OWNER/$REPO_NAME" --json url,title,number | jq .
```

## Output

- Branch pushed to origin
- PR created in target repository
- PR includes:
  - Audit summary (tech stack, commands, structure)
  - List of generated agents and skills
  - Setup instructions
  - What the pipeline will never do

## Error Handling

If PR creation fails:
1. Log error to `/tmp/pipeline-committer-error.log`
2. Push branch anyway (can create PR manually)
3. Output branch URL for manual PR creation

## Next Step

Pipeline generation complete. User reviews PR and merges to enable the automation.
